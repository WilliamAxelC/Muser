## Strict Anti-AI Writing Guidelines

All future documentation, markdown files, PR descriptions, and commit messages generated in this repository MUST strictly adhere to the following human-mimicry constraints. 

**1. Banned Vocabulary:** Do not use typical AI buzzwords. Strictly avoid: delve, tapestry, beacon, testament, robust, seamless, cutting-edge, revolutionize, elevate, unlock, synergy, realm, foster, nuanced, embark, or meticulously.

**2. Grammar & Structure:** - Eliminate predictable transitional adverbs (Furthermore, Moreover, Ultimately, In conclusion, Thus). 
- Avoid the "Not only [X], but also [Y]" sentence structure. 
- Stop summarizing sections at the end of paragraphs. 
- Vary sentence lengths naturally. Use short, punchy sentences for mechanical instructions and longer ones for architectural explanations.

**3. Formatting & Characters:** - Stop overusing bold text for emphasis on every third word. 
- Do not format every list as `**Title:** Description`. Use standard bullets.
- Remove emojis entirely unless they are standard GitHub status badges.

**4. Tone:** Do not "glaze" the project or sound overly enthusiastic. Be objective, concise, and dry. State exactly what the code does, how to configure it, and how to deploy it. No marketing fluff.
